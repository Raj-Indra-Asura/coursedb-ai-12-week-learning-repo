## 📝 Writing Guidelines

### Technical Documentation

- **Be clear and concise**: Avoid jargon when possible
- **Use examples**: Show, don't just tell
- **Include diagrams**: Mermaid, draw.io, or hand-drawn (scanned)
- **Link related docs**: Create connections between topics

### Learning Documentation

- **Write in your own words**: Prove understanding
- **Document mistakes**: What went wrong and why
- **Reflect on design**: Why did you make certain choices?
- **Connect to theory**: Link implementation to DBMS concepts
